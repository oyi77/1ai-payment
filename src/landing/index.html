FAQ

Pertanyaan yang mungkin Anda punya Questions you'll have

Apa yang terjadi jika gateway mati?What happens if a gateway goes down?
Kami tidak terikat pada satu gateway — integrasi Anda gateway-agnostic. Jika satu provider mengalami masalah, ubah parameter gateway di payment request Anda dan transaksi dialihkan ke backup. Kode checkout Anda tidak pernah berubah. Kami juga sedang mengerjakan automatic fallback chain untuk rilis mendatang. We don't terminate at a single gateway — your integration is gateway-agnostic. If one provider has an outage, change the gateway parameter in your payment request and transactions route through a backup. Your checkout code never changes. We're also working on automatic fallback chains for future releases.
Di mana data transaksi disimpan, dan siapa yang bisa mengaksesnya?Where is transaction data stored, and who can access it?
Data transaksi disimpan di database LibSQL pada infrastruktur kami, terbatas per merchant. Kami mencatat data event yang sudah dinormalisasi (order_id, status, amount) bukan payload gateway mentah. API key setiap merchant hanya memberikan akses ke data mereka sendiri. Kami tidak pernah menyimpan API key atau secret gateway dalam bentuk plaintext. Transaction data is stored in a LibSQL database on our infrastructure, scoped per merchant. We log normalized event data (order_id, status, amount) not raw gateway payloads. Each merchant's API key provides access only to their own data. We never store gateway API keys or secrets in plaintext.
Apakah ini relevan dengan PCI, dan apa yang masuk/keluar dari scope?Is this PCI-relevant, and what's in/out of scope?
1ai-payment tidak menyimpan, memproses, atau mentransmisikan nomor kartu (PAN). Data pembayaran langsung dari pelanggan ke gateway — kami hanya menerima payment URL dan status callback. Platform kami hanya menangani lapisan agregasi. PCI SAQ A scope untuk sebagian besar merchant, tapi konsultasikan dengan QSA Anda untuk implementasi spesifik. 1ai-payment does not store, process, or transmit card numbers (PANs). Payment data goes directly from the customer to the gateway — we only receive payment URLs and status callbacks. Our platform handles the aggregation layer only. PCI SAQ A scope for most merchants, but consult your QSA for your specific implementation.
Bisakah saya white-label atau menjual kembali ini untuk klien?Can I white-label or resell this for clients?
Ya — itulah mengapa model API key merchant-scoped dibuat. Deploy infrastruktur pembayaran per klien dengan rate limit, riwayat transaksi, dan endpoint webhook yang terisolasi. Hubungi kami untuk agency pricing dan paket bertingkat. Yes — that's what the merchant-scoped API key model is built for. Deploy payment infrastructure per client with isolated rate limits, transaction history, and webhook endpoints. Contact us for agency pricing and tiered plans.
