How it works

Payments in 3 steps

No SDKs, no complex onboarding. Just HTTP requests to a unified API.

01
💳

Create Payment

POST /api/payments with amount, currency, gateway. Returns a payment URL in milliseconds.

02
🎯

Customer Pays

Redirect your customer to the gateway URL. They complete payment on their preferred method.

03
📬

We Forward It

Gateway callback hits our webhook. We verify signature, normalize, and forward to your callback URL with retries.

FAQ

Questions you'll have

What happens if a gateway goes down?
We don't terminate at a single gateway — your integration is gateway-agnostic. If one provider has an outage, change the gateway parameter in your payment request and transactions route through a backup. Your checkout code never changes. We're also working on automatic fallback chains for future releases.
Where is transaction data stored, and who can access it?
Transaction data is stored in a LibSQL database on our infrastructure, scoped per merchant. We log normalized event data (order_id, status, amount) not raw gateway payloads. Each merchant's API key provides access only to their own data. We never store gateway API keys or secrets in plaintext.
Is this PCI-relevant, and what's in/out of scope?
1ai-payment does not store, process, or transmit card numbers (PANs). Payment data goes directly from the customer to the gateway — we only receive payment URLs and status callbacks. Our platform handles the aggregation layer only. PCI SAQ A scope for most merchants, but consult your QSA for your specific implementation.
Can I white-label or resell this for clients?
Yes — that's what the agency/merchant-scoped API key model is built for. Deploy payment infrastructure per client with isolated rate limits, transaction history, and webhook endpoints. Contact us for agency pricing and tiered plans.
What's the actual cost at scale?
Free tier: 30 req/min, standard per-transaction fees (2.5% + gateway costs). Pro: 120 req/min, 2% fee. Enterprise: custom rate limit, 1% or flat-rate, dedicated support. The aggregation layer doesn't add gateway markup beyond our fee. Contact us for volume projections and a cost comparison vs. direct integrations.
Who operates this — is there support if something breaks in production?
Operated by the BerkahKarya team (CV Graha Insan Mandiri Berkah). We maintain the infrastructure, handle gateway API changes, and provide support via the dashboard and direct channels for Pro/Enterprise merchants. The systemd-managed service includes auto-restart, journald logging, and health checks.
